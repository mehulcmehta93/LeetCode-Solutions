# Two Sum

## Pattern
HashMap

## Difficulty
Easy

## Brute Force
Time: O(n²)
Space: O(1)

## Optimized
Time: O(n)
Space: O(n)

## Key Idea
Store previously seen numbers in a HashMap so complement lookup is O(1).

## Interview Questions
- Why HashMap?
- Why not HashSet?
- What if there are duplicate numbers?
